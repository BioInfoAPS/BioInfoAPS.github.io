Transferring files

Transfering files to and from remote server and your local computer requires a separate transfer protocol, such as Samba or SFTP. The complete list of data transfer methods available for HiperGator is listed in its wiki page. We recommend use of SFTP (Secure File Transfer Protocol) for accessing files for this workshop. Although commandline version of SFTP are often bundled with SSH, we recommend use of graphical softwares for ease of use if you are not familiar with sftp commands.

  • Windows
  • MacOS
  • Linux

You can use Filezilla and WinSCP for uploading or downloading data from remote SFTP server.

You can use Filezilla and Cyberduck for uploading or downloading data from remote SFTP server.

You can mount SFTP as virtual filesystem using sshfs and access the files using file any file manager. 

sudo apt install sshfs
mkdir /<mountpoint>
sshfs <username>@hpg.rc.ufl.edu:/blue/general_workshop/<username> /opt/<mountpoint>

Alternatively, you can use Filezilla for uploading or downloading data from remote SFTP server.


The following instructions are for Filezilla.

  1. Download Filezilla using this link.
  2. Install Filezilla and then run the program.
  3. Fill the following information in quickconnect panel and press Quickconnect button.
    • Host: hpg.rc.ufl.edu
    • Username: provided username
    • Password: provided passsowd
    • Port: 22
  4. Click OK if prompted to certify the RSA key fingerprint.
  5. Change remote site to the following. Do not forget to replace <username> with the provided username. 
    /blue/general_workshop/<username>
